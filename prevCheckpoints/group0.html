  1. sudo /sbin/service httpd status
  2. ServerRoot is the top directory of the server (/etc/httpd). DocumentRoot is the directory where the .html files displayed on the webpage exist (/var/www/html).
  3. 80
  4. The DocumentRoot directory (/var/www/html)
  5. Traffic Logs: /etc/httpd/logs/access_log. Error Logs: /etc/httpd/logs/error_log
  6. The Directory Index File is the default file that is displayed when you navigate to a certain directory. It's nice to have one because you can control what is seen when you navigate to that directory rather than having a list of all files in the directory.
  7. In the httpd.conf file, you can enable public_html by commenting out "#disable UserDir public_html" and uncommenting "enable UserDir public_html".
  8. To create a password protected directory, you first need to create a .htpasswd file which you can do using the command "htpasswd -c [directory] [username]. Then, you need to make a .htaccess file that looks like the following:
    AuthType Basic
    AuthName "Secure Files"
    AuthUserFile [directory to .htpasswd]
    Require valid-user
    After that, place the .htaccess file into the directory you want to password protect. When trying to access that directory via a browser, it should be password protected along with its subfolders.
  9. To send the browser to a particular document upon issuing a 404 error code, use the ErrorDocument directive. In the case of a 404 error, type ErrorDocument 404 -URL to redirect to- in the conf file.
  10. GET and HEAD both request data from a specified resource, but HEAD only returns HTTP headers and no document body. Both only retrieve data. POST submits data to be processed to a specified resource.
